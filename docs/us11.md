# US11 — Suporte Multi-edge

## Runtimes alvo
- Cloudflare Workers
- Vercel Edge
- Azion
- Fastly
- Fallback: `unknown`

## Abordagem
- Detectar provider por heurísticas simples (variáveis globais/ambiente).
- Normalizar `Request` para uma forma comum `{ url, method, headers, bodyString }`.
- Normalizar `Response` para `{ body, status, headers }`, sem depender de APIs proprietárias.
- Wrapper `edgeHandler` detecta o provider, normaliza request/response e invoca a pipeline única.

## Limitações
- Heurísticas básicas podem requerer ajuste em produção para atributos específicos de cada runtime.
- O wrapper não cobre APIs avançadas (KV, caches), apenas request/response base.
