## 2. Objetivo da alteração
Criar exemplos reais de configuração para destravar os próximos US (US02/US03) pedindo 4 números de WhatsApp e 16 mensagens randomizadas. Garantir que `config/index.js` já exponha arrays prontos para seleção de números/mensagens, mantendo rastreabilidade com a arquitetura 0.1.0. Escopo inclui apenas geração dos valores de exemplo e atualização da configuração; não inclui lógica de seleção, encoding ou documentação específica dos US.

## 3. Decisões e justificativas
- **Estrutura única vs seções aninhadas:** Avaliei criar um objeto `whatsapp` aninhando números/mensagens, mas preferi manter arrays no topo do config para casar com as tasks T03-001/T03-003 (que citam `messages` diretamente). Isso reduz atrito quando as funções de seleção forem implementadas.
- **Valores pseudoaleatórios vs placeholders previsíveis:** Optei por gerar valores pseudoaleatórios (E.164 válidos + strings hifenizadas) em vez de sequências “1234…”, para simular dados reais e permitir testes de randomização. Isso também atende ao pedido explícito do usuário.
- **Risco:** Arrays ainda não são usados por nenhuma função; dependerão das próximas tasks. Mitigação: documentado nos próximos passos e alinhado com US02/US03.

## 4. Implementação
- **Arquivo alterado:** `config/index.js` agora contém `numbers` com 4 entradas em formato E.164 e `messages` com 16 strings únicas, preservando `architectureVersion`.
- **Notas:** números e mensagens foram gerados manualmente de forma aleatória, mantendo apenas caracteres seguros para futura interpolação (sem espaços, evitando encoding imediato). Nenhum script adicional ou teste automático necessário, pois não há lógica executável vinculada ainda.
